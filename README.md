# LegacyRegion

LegacyRegion — плагин приватных регионов на блоках для Lumi. Игрок ставит настроенный блок региона, а плагин автоматически создает вокруг него защищенную область с владельцем, участниками, меню управления, визуальными эффектами и гибкой настройкой поведения.

Главная идея плагина — дать серверу максимально кастомизируемые регионы без привязки к одному конкретному блоку. Можно подключать разные типы регионов, задавать им радиус, название, иконку в меню, правила разрушения блока региона и внешний вид сообщений.

## Возможности

- Несколько типов регионов через отдельные файлы `regions/*.yml`.
- Подключение своих блоков региона через `namespace-id`, `legacy-id` и `damage`.
- Включение и отключение каждого типа региона через `enabled`.
- Индивидуальный радиус для каждого блока региона.
- Иконки кнопок регионов в меню из ванильного текстурпака Minecraft.
- Возможность включать и отключать иконку через `button-icon-enabled`.
- Отдельная настройка, может ли блок региона ломаться от взрывов.
- Меню управления регионами через FormConstructor.
- Добавление и удаление участников региона.
- Передача владельца региона через меню настроек.
- Настройка, могут ли участники ломать сам блок региона.
- Возможность скрывать координаты региона в меню.
- Защита территории от чужого строительства, ломания и взаимодействий.
- Настраиваемый обход защиты для операторов.
- Защита от пересечения регионов.
- Настраиваемые частицы при установке региона и отказе действия.
- Голограммы над блоками регионов.
- SQLite и MySQL хранилище.
- Полная настройка сообщений в отдельном `messages.yml`.
- Переключение формата сообщений между чатом и toast-уведомлениями простым переносом ключа.
- Отдельный `menus.yml` для текста всех кнопок и форм меню.

## Сборка

```powershell
.\gradlew.bat build
```

Для копирования jar в папку сервера:

```powershell
.\gradlew.bat copyToServer
```

`FormConstructor.jar` и `JOOQConnector-Nukkit-1.0.1.jar` должны лежать рядом с плагином в папке `plugins`.

## Настройки

- `config.yml` хранит настройки команд, защиты, частиц, голограмм и регистрацию блоков регионов.
- `messages.yml` хранит сообщения. Ключи можно переносить между `chat-message` и `toast-message`, чтобы выбрать вывод в чат или toast-уведомление.
- `menus.yml` хранит текст кнопок и форм меню.
- `regions/*.yml` описывает типы регионов. `enabled` включает или отключает тип региона, `button-icon-enabled` управляет показом иконки, `button-icon` задает иконку кнопки в меню, `break-from-explosions` разрешает или запрещает ломать блок региона взрывами.
